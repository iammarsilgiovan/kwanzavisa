## 🧭 CONTEXTO DO PROJECTO

Estás a construir **KwanzaVisa**, uma plataforma fintech angolana que permite a qualquer angolano aceder a serviços financeiros e de compras internacionais pagando em Kwanzas.

**Serviços da plataforma:**
- 💳 **Cartão Virtual** — gerar cartão virtual para pagamentos online
- 🖱️ **Acesso Assistido** — o cliente usa as suas próprias plataformas com o nosso cartão/suporte (novo nome para o serviço antes chamado "pagamento assistido" — ver definição abaixo)
- 🔁 **Transferência com Liquidação Local** — envio de valores entre países
- 🌍 **Abertura de Conta Internacional** — abertura de conta em Wise, Bybit ou Kast (orçamento sob consulta)

> **Utilizadores-alvo:** Angolanos que precisam pagar serviços internacionais, fazer compras online (Amazon, Shein, Alibaba, etc.) ou ter uma conta financeira internacional, usando Kwanzas.

## 📖 DEFINIÇÃO DOS SERVIÇOS

### 💳 Cartão Virtual Internacional
O cliente solicita um cartão virtual KwanzaVisa para usar em pagamentos online internacionais. O cartão é gerado e entregue via WhatsApp com os dados necessários para uso imediato.

### 🖱️ Acesso Assistido ← NOME OFICIAL
> **Definição:** Serviço em que o cliente usa as suas próprias contas e plataformas digitais — a KwanzaVisa fornece o método de pagamento (cartão) e o suporte técnico para que o cliente conclua a compra de forma autónoma. O cliente mantém total controlo da sua conta; nós facilitamos o pagamento.

**Exemplos de uso:** Comprar na Amazon, Shein, Alibaba, AliExpress, pagar Netflix, Spotify, domínios, software, cursos online.

**Copywriting para o site:**
- "Tu fazes a compra, nós tratamos do pagamento."
- "A tua conta. O nosso cartão. A tua independência."

### 🔁 Transferência com Liquidação Local
Envio e recepção de valores entre países sem transferência bancária directa. O remetente paga em Kz, o destinatário recebe na moeda local do país de destino.

> ⚠️ Nunca usar "remessa internacional" ou "envio internacional directo". Usar sempre "transferência com liquidação local" ou "entrega de valores entre países".

### 🌍 Abertura de Conta Internacional
Serviço de apoio à abertura de conta em plataformas financeiras internacionais. Disponível para:

| Plataforma | Descrição |
|------------|-----------|
| **Wise**   | Conta multi-moeda europeia, ideal para receber pagamentos internacionais |
| **Bybit**  | Conta em exchange cripto com funcionalidades de pagamento |
| **Kast**   | Cartão e conta virtual para utilizadores sem acesso bancário tradicional |

**Modelo de preço:** Orçamento sob consulta (sem preço fixo no site). O cliente submete pedido → admin contacta via WhatsApp com orçamento personalizado.

**Copywriting:**
- "Abre a tua conta internacional hoje. Nós tratamos de tudo."
- "Wise, Bybit ou Kast — escolhe a plataforma, nós fazemos o resto."

## 💱 SISTEMA DE CÂMBIO EM TEMPO REAL

### Lógica de cálculo
```
câmbio_final_usd = câmbio_mercado_usd + 150    (Kz por USD)
câmbio_final_eur = câmbio_mercado_eur + 190    (Kz por EUR)

valor_kz = valor_input × câmbio_final

Exemplo: mercado USD = 900 Kz → câmbio público = 1.050 Kz/USD
```

> ⚠️ A margem (150 Kz / 190 Kz) é o lucro da empresa. **Nunca exposta ao utilizador.**

### API de câmbio em tempo real
```
Usar a API gratuita: https://api.exchangerate-api.com/v4/latest/USD
(ou alternativa: https://open.er-api.com/v6/latest/USD)

Fluxo automático:
1. Cloud Function agendada (Firebase Scheduler) → executa a cada 30 minutos
2. Faz GET à API de câmbio externo
3. Obtém USD/AOA e EUR/AOA (Kwanza angolano)
4. Aplica margem: +150 Kz (USD), +190 Kz (EUR)
5. Guarda em Firestore na colecção 'configuracoes':
     cambio_usd_mercado, cambio_eur_mercado,
     cambio_usd_final, cambio_eur_final,
     ultima_actualizacao (timestamp)

No frontend:
  → Simulador lê 'configuracoes' em tempo real (Firestore onSnapshot)
  → Sempre mostra câmbio actualizado sem refresh

Admin pode forçar actualização manual com botão [Actualizar câmbio agora]
Admin pode sobrescrever temporariamente o câmbio (override manual)
```

### Ecrã do simulador no frontend
